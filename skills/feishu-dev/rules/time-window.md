# 时间窗口推断规则

> feishu-dev Phase 1.8c 的结构化依据。
> 目标：从任务上下文推断 bug 发生时间，构造查日志的 [start, end] 窗口。

---

## 时间线索优先级

| 优先级 | 来源 | 示例 |
|--------|------|------|
| **P1 最高** | 截图中可见的时间信息 | 系统时间栏、日志时间戳、界面上的时间 |
| **P2** | 任务描述里的时间短语 | "今天下午两点"、"昨晚"、"刚才"、"上午十点左右" |
| **P3 兜底** | `task.created_at`（任务创建时间） | 无其他线索时使用 |

---

## 窗口计算规则

| 线索类型 | 前推窗口 | 后推缓冲 | 示例 |
|---------|---------|---------|------|
| "刚才"、"刚刚"、"刚好" | 30 分钟 | +15 min | 刚才报错 → `[now-30m, now+15m]` |
| "今天上午"、"今天下午" | 4 小时 | +15 min | 今天下午 → `[T-4h, T+15m]` |
| 具体时间点（"14:30"） | 1 小时 | +15 min | 14:30 出错 → `[13:30, 14:45]` |
| "昨天"、"昨晚" | 用昨日 20:00-23:59 | — | 昨晚 → `[昨日20:00, 昨日23:59]` |
| 仅 task.created_at（fallback） | 1 小时 | +15 min | 创建时间 T → `[T-1h, T+15m]` |

---

## ISO 8601 格式要求

查询时间必须是带时区的 ISO 8601 格式：

```
正确：2026-06-04T13:30:00+08:00
正确：2026-06-04T05:30:00Z
错误：2026/06/04 13:30       ← GUANCE_TIME_INVALID
错误：2026-06-04 13:30       ← GUANCE_TIME_INVALID
```

推断出的时间点 → `+08:00` 结尾（假设服务器在北京时区）。

---

## 输出要求

```
bug_start = <ISO8601+08:00>
bug_end   = <ISO8601+08:00>
```

推断过程可选输出（辅助调试）：
```
[1.8c] 时间推断：线索="<摘录>"，时间点=<YYYY-MM-DDTHH:MM>，窗口=<-Xh/+15min>
```

---

## 调参场景

**当截图时间和描述时间冲突**：优先截图（P1 > P2）。

**无任何线索**：用 `task.created_at`，窗口 ±1h，并在 1.8d 的 log 输出标注 `(fallback: created_at)`。

**跨天的 bug**（如"前天出现过"）：把窗口扩展到当天全天（00:00 ~ 23:59）。
