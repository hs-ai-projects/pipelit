22.## pipelit /release skill 改进建议（来自 v3.9.3 发版复盘）

### 问题 1：收集 commit 信息时丢失 body（最高优先级）

**现象**：Phase 1 用 `--oneline` 收集 commit，导致 commit body 里的 `Feishu-Task` 链接全部丢失，changelog 只有一个飞书链接。

**修复**：Phase 1.3 改用完整格式收集 commit：
git log <last_tag>..HEAD --format="=== %h ===%n%B" --no-merges
git log <last_tag>..HEAD --format="=== %h ===%n%B" --merges

解析时从 body 提取 `Feishu-Task: <url>` 字段，直接作为该条 changelog 的飞书链接，不再做语义匹配。

---

### 问题 2：卡片内容没有预览确认步骤（高优先级）

**现象**：Phase 3 直接发卡片，用户只能发完才知道内容对不对，本次因此重发了 6 次。

**修复**：在 Phase 3 发送前增加一个预览确认步骤，展示：
- 每条 changelog 的完整文字
- 每条对应的飞书任务链接
- 每条 @ 的人（过滤黑名单后）

用户确认后再发，或直接修改内容再发。


---

### 问题 4：发版前无法预览 @ 关注人列表

**现象**：只有发出去才知道 @ 了谁，follower 在黑名单里时无法提前感知。

**修复**：建议新增 `preview_task_mentions` 子命令，在发卡片前打印所有 task 的 follower 查询结果（含黑名单过滤后的最终 @ 人），让用户确认后再发。
